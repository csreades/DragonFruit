# Raft System Documentation

## Overview

The raft system provides a dynamically-generated sacrificial base platform for 3D printing supports. It creates a flat, stable foundation that improves build plate adhesion and prevents support bases from warping or failing during the print. The raft features chamfered edges for easy removal and crenelations (suction relief slots) to prevent vacuum adhesion to the FEP film in resin printers.

## Goals and Design Philosophy

### Primary Goals
1. **Build Plate Adhesion**: Provide a large, flat surface area that adheres strongly to the build plate
2. **Support Stability**: Create a unified platform that connects all support bases, preventing individual supports from failing
3. **Easy Removal**: Chamfered edges allow tools to slide under the raft for clean separation from the build plate
4. **Suction Relief**: Crenelations break the vacuum seal between the raft and FEP film during layer separation in resin printers
5. **Material Efficiency**: Minimize resin usage while maintaining structural integrity

### Design Decisions

#### Chamfered Edges (45° Default)
- **Purpose**: Creates a beveled edge that tools can easily slide under
- **Angle Range**: 45-90° (adjustable)
- **Implementation**: Bottom surface is offset inward from top surface by `thickness * tan(90° - angle)`
- **Benefit**: Prevents sharp corners that would be difficult to remove and could damage the build plate

#### Crenelated Perimeter Wall
- **Purpose**: Rectangular gaps in the wall break the vacuum seal during layer separation
- **Placement**: One centered gap per straight edge only (curved edges remain solid)
- **Height**: 0.35mm default (sits on top of base raft)
- **Thickness**: 0.5mm default (provides structural support)
- **Gap Detection**: Straight edges identified by angle tolerance (≤15° between consecutive polygon edges)
- **Manifold Geometry**: Each gap has perpendicular side faces to ensure watertight mesh

#### Dynamic Footprint
- **Computation**: Convex hull of all support base circles with small margin (0.2mm)
- **Sampling**: 24 points per circle for smooth approximation
- **Algorithm**: Monotonic chain convex hull (O(n log n))
- **Auto-Update**: Raft regenerates whenever supports are added, moved, or removed

## Technical Implementation

### Architecture

The raft system follows a modular, domain-first architecture:

```
src/supports/Rafts/Crenelated/
├── RaftTypes.ts              # Type definitions
├── RaftDefaults.ts           # Default settings
├── RaftState.ts              # State management
├── geometry/
│   ├── computeFootprint.ts           # Convex hull from support bases
│   ├── generateChamferedBase.ts      # Base plate with chamfered edges
│   ├── generatePerimeterWall.ts      # Solid wall (fallback)
│   ├── generateCrenelatedWallManual.ts # Crenelated wall with gaps
│   ├── insetConvexPolygon.ts         # Polygon offset utility
│   └── utils.ts                      # Geometry helpers
├── rendering/
│   └── RaftRenderer.tsx      # 3D rendering component
└── components/
    └── RaftSettingsCard.tsx  # UI controls
```

### Geometry Generation Pipeline

#### 1. Footprint Computation (`computeFootprint.ts`)
```typescript
Input: Support base circles (center, radius)
Process:
  1. Sample each circle perimeter (24 points)
  2. Compute convex hull using monotonic chain algorithm
  3. Add small margin (0.2mm) for coverage
Output: 2D polygon (array of Vector2)
```

#### 2. Chamfered Base Generation (`generateChamferedBase.ts`)
```typescript
Input: Footprint polygon, thickness, chamfer angle
Process:
  1. Top profile = footprint at Z = thickness
  2. Bottom profile = inset footprint at Z = 0
     - Inset distance = thickness * tan(90° - angle)
  3. Build geometry:
     - Top face (horizontal)
     - Bottom face (horizontal)
     - Side faces (angled, connecting top to bottom)
  4. Validate vertices (no NaN values)
  5. Compute vertex normals
Output: THREE.Mesh (base plate)
```

#### 3. Crenelated Wall Generation (`generateCrenelatedWallManual.ts`)
```typescript
Input: Footprint polygon, wall settings
Process:
  1. Detect straight edges:
     - Compute edge directions and lengths
     - Mark edges where angle change < 15°
     - Require minimum 5mm length
  2. For each edge:
     - If curved: build solid wall segment
     - If straight: build wall with 1 centered gap
  3. Gap placement:
     - Center gap in usable length (excluding 1mm margins)
     - Width = crenulationGapWidth setting
     - Depth = wallThickness (fully penetrates wall)
  4. Build geometry manually:
     - Outer face, inner face, top face, bottom face
     - Perpendicular side faces at each gap opening
  5. Compute vertex normals
Output: THREE.Mesh (perimeter wall)
```

### Straight Edge Detection

The system uses a robust edge-based detection algorithm:

1. **Edge Analysis**: For each polygon edge, compute direction vector and length
2. **Angle Comparison**: Compare consecutive edge directions using dot product
3. **Threshold**: Edges are "straight" if angle change ≤ 15° and length ≥ 5mm
4. **Run Grouping**: Consecutive straight edges form a "straight run"
5. **Crenelation Placement**: One gap placed at the center of each straight run

This approach handles discretized convex hulls correctly, identifying truly flat sides while keeping curved sections solid.

### Manual Geometry Construction

Unlike the initial Shape/ExtrudeGeometry approach (which created artifacts), the final implementation builds geometry manually:

```typescript
function addQuad(p0, p1, p2, p3, flip) {
  // Add 4 vertices
  positions.push(p0.x, p0.y, p0.z, ...)
  
  // Add 2 triangles with correct winding
  if (flip) {
    indices.push(base, base+2, base+1, base, base+3, base+2)
  } else {
    indices.push(base, base+1, base+2, base, base+2, base+3)
  }
}
```

**Benefits:**
- Complete control over vertex positions and normals
- Gaps are true voids (no geometry at all)
- Perpendicular side faces ensure manifold mesh
- No triangulation artifacts from complex shapes

### State Management

**RaftState.ts** provides a simple store pattern:
- `getRaftSettings()`: Returns current settings
- `setRaftSettings(settings)`: Updates settings and notifies subscribers
- `subscribeToRaftStore(callback)`: Registers change listener

**Integration:**
- `RaftRenderer.tsx` subscribes to both support store and raft store
- Auto-regenerates geometry when either changes
- Uses `useMemo` to cache expensive geometry calculations

### Rendering Integration

**RaftRenderer.tsx:**
1. Subscribes to support list and raft settings
2. Filters supports to get base circles
3. Computes footprint from circles
4. Generates base mesh (chamfered)
5. Generates wall mesh (crenelated or solid)
6. Applies materials (blue base, green wall for testing)
7. Returns THREE.Group containing both meshes

**Scene Integration:**
- Rendered in `SceneCanvas.tsx` alongside supports
- Support base discs hidden when raft enabled
- Raft positioned at Z=0 (build plate level)

## User Interface

### Raft Settings Card

Located in support mode sidebar, provides controls for:

1. **Enable Raft**: Toggle checkbox (default: enabled)
2. **Raft Thickness**: 0.1-2.0mm slider + text input (default: 0.5mm)
3. **Edge Chamfer Angle**: 45-90° slider + text input (default: 45°)
4. **Wall Height**: 0.1-1.0mm slider + text input (default: 0.35mm)
5. **Wall Thickness**: 0.2-2.0mm slider + text input (default: 0.5mm)

**UI Features:**
- Collapsible header with eye icon
- Dual input: sliders for quick adjustment, text fields for precise values
- Real-time preview updates in 3D scene
- Input validation with range clamping

**Removed Controls:**
- Crenulation Gap Width (now fixed at 1.5mm)
- Crenulation Spacing (now 1 gap per straight edge)

These were removed to simplify the UI since the fixed-count approach works well.

## Default Settings

Optimized for resin printing:

```typescript
{
  enabled: true,
  thickness: 0.5,        // mm - thin but strong
  chamferAngle: 45,      // degrees - easy removal
```

## Performance Considerations

### Optimization Strategies
1. **Memoization**: Geometry only regenerates when dependencies change
2. **Efficient Hull**: Monotonic chain algorithm is O(n log n)
3. **Manual Geometry**: Avoids expensive Shape triangulation
4. **Selective Updates**: Only affected components re-render

### Typical Performance
- Footprint computation: <1ms for 10 supports
- Base mesh generation: <5ms
- Wall mesh generation: <10ms
- Total regeneration: <20ms (imperceptible)

## Future Enhancements

### Potential Improvements
1. **Variable Gap Count**: UI control for gaps per edge (1-5)
2. **Gap Width Control**: Re-expose in UI if users want customization
3. **Raft Patterns**: Optional grid or honeycomb infill for large rafts
4. **Multi-Level Rafts**: Stepped raft for tall support clusters
5. **Smart Margins**: Adaptive margin based on support density
6. **Export Options**: Separate raft STL for multi-material printing

### Known Limitations
1. **Convex Only**: Raft is always convex (no concave footprints)
2. **Fixed Gap Count**: Currently hardcoded to 1 per edge
3. **Straight Detection**: May miss very subtle curves in discretized polygons
4. **No Infill**: Solid base only (no internal structure options)

## Troubleshooting

### Common Issues

**Raft not appearing:**
- Check "Enabled" toggle in Raft Settings
- Ensure at least one support exists
- Verify raft thickness > 0

**Gaps in wrong places:**
- Straight edge detection may be too sensitive
- Adjust `straightTolDeg` in `generateCrenelatedWallManual.ts`
- Check polygon has sufficient vertices for smooth curves

**Mesh artifacts:**
- Ensure normals are computed after geometry creation
- Validate no NaN values in vertex positions
- Check winding order for all faces

**Performance issues:**
- Reduce `samplesPerCircle` in `computeFootprint` (default: 24)
- Use fewer supports or smaller base radii
- Check for unnecessary re-renders in React components
