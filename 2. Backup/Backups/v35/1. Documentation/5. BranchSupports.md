# Branch Supports - Complete Technical Documentation

## Overview

Branch supports are a specialized support type that attach to existing supports (trunks or other branches) rather than the build plate. They enable tree-like support structures where branches can connect to branches, creating efficient, material-saving support networks.

**Key Characteristics:**
- Contact cone tip touches the model surface
- Base attaches to parent support shaft (not build plate)
- Connected via special "branch joint" that locks to parent shaft
- Can form multi-level trees: trunk → branch → branch → branch...
- No base disc (only trunks have base discs)

4. **Shaft Projection**: Branch joints constrained to slide only along parent shaft axis

## Placement Workflow

### State Machine

Branch placement uses a three-stage state machine:

```typescript
type BranchPlacementStage = 'idle' | 'tipPlaced' | 'baseFollow';

interface BranchPlacementState {
  stage: BranchPlacementStage;
  contact?: { point: Vec3; normal: Vec3 };  // First click: model contact
  snap?: { trunkId: string; position: Vec3 }; // Snapped to parent shaft
}
```

**Stages:**

1. **`idle`**: Waiting for Alt+click on model
2. **`tipPlaced`**: Contact cone placed, base following mouse in 3D
3. **`baseFollow`**: (same as tipPlaced, just tracking mouse)

### User Interaction

**Step 1: First Click (Alt+Click on Model)**
- User holds Alt key and clicks on model surface
- Places contact cone at click point with surface normal
- Enters `baseFollow` mode
- Base position follows mouse in 3D space

**Step 2: Base Tracking**
- Mouse position projected from screen to 3D world space
- Uses camera unproject at contact cone Z depth
- Base moves freely in 3D until near a support shaft
- When within 5mm of shaft: snaps to shaft centerline (color changes)

**Step 3: Second Click (Finalize)**
- User clicks anywhere while snapped to valid support
- Creates branch support instance with 3 joints
- Resets to `idle` state

### Snapping Logic

**File:** `src/supports/BranchSupports/snapping/snapToTrunk.ts`

```typescript
export function snapToTrunk(
  mouse: Vec3,
  supports: SupportInstance[],
  snapDistanceMm: number = 3.0,
  cameraPosition?: Vec3
): { trunkId: string; position: Vec3 } | null
```

**Algorithm:**
1. Iterate through all supports (trunks and branches)
2. Build shaft segments for each support:
   - **Trunks with joints**: Check all segments between consecutive joints, plus final segment to base (if last joint isn't branch type)
   - **Trunks without joints**: Single segment from tip socket → base top
   - **Branches**: Check all segments from tip joint through middle joints to branch joint
3. For each segment, project mouse position onto shaft centerline
4. Use depth-aware prioritization when camera position available:
   - If support is significantly closer to camera (>1mm depth difference), prefer it
   - If depths are similar (within 1mm), prefer closer distance to mouse
   - Must be within snap threshold (3mm by default)
5. Return best snap position or null

**Key Details:**
- **Snap distance**: 3mm (reduced from 5mm for precision in dense areas)
- **Depth-aware**: Prioritizes supports closest to camera when overlapping visually
- **Multi-segment**: Checks all shaft segments, not just endpoints
- **Centerline snapping**: Snaps to shaft centerline (not surface)
- **No phantom projections**: Branches limited to tip joint → branch joint
- **Camera integration**: Camera position passed from SceneCanvas through page.tsx to hook

**Depth Prioritization Logic:**
```typescript
function shouldUpdateBest(
  newDist: number,
  newPos: Vec3,
  bestDist: number,
  bestDepth: number,
  snapThreshold: number,
  cameraPos?: Vec3
): boolean
```
- Calculates distance from camera to snap position
- If new position is >1mm closer to camera, prefer it (even if slightly farther from mouse)
- If depths within 1mm, use standard distance comparison
- Ensures user can select visually front-most support in crowded scenes

### Joint Configuration

Branch supports have **3 joints** by default:

#### 1. Tip Joint (Order 0)
```typescript
{
  id: string;
  position: Vec3;           // At contact cone base (socket face)
  ballDiameterMm: number;   // shaft diameter + 0.1mm
  order: 0;
  isTipJoint: true;
  type: 'standard';
  updatedAt: number;
}
```
- Gray ball joint
- Located at base of contact cone (where shaft starts)
- Movable freely in 3D space

#### 2. Middle Joint (Order 1)
```typescript
{
  id: string;
  position: Vec3;           // Halfway between tip joint and branch joint
  ballDiameterMm: number;   // shaft diameter + 0.1mm
  order: 1;
  isTipJoint: false;
  type: 'standard';
  updatedAt: number;
}
```
- Gray ball joint
- Positioned at midpoint of shaft
- Allows user to adjust branch angle
- Movable freely in 3D space

#### 3. Branch Joint (Order 2)
```typescript
{
  id: string;
  position: Vec3;           // At parent shaft connection
  ballDiameterMm: number;   // PARENT shaft diameter + 0.1mm (larger!)
  order: 2;
  isTipJoint: false;
  type: 'branch';           // Special type!
  lockedToSupportId: string; // Parent support ID
  updatedAt: number;
}
```
- **Yellow ball joint** (visual distinction)
- **Larger diameter** than branch shaft (matches parent shaft + 0.1mm)
- **Locked to parent shaft** - can only slide along parent axis
- **Cannot move off shaft** - constrained by projection algorithm

### Constraint System

**File:** `src/supports/BranchSupports/constraints/branchJointConstraint.ts`

#### Core Function: `constrainBranchJointToShaft()`

```typescript
export function constrainBranchJointToShaft(
  desiredPosition: Vec3,
  branchJoint: SupportJoint,
  parentSupport: SupportInstance
): Vec3
```

**Algorithm:**

1. **Calculate Parent Shaft Endpoints**
   ```typescript
   // Shaft starts at tip socket
   const shaftStart = tip + tipNormal * tipLength;
   
   // Shaft end depends on parent type
   if (isBranch(parent)) {
     shaftEnd = parent.branchJoint.position;  // Branch parent
   } else {
     shaftEnd = base + baseHeight;            // Trunk parent
   }
   ```

2. **Build Shaft Segments**
   - If no joints: single segment from shaftStart to shaftEnd
   - If joints exist: build polyline through all joints
   - Segments: shaftStart → joint0 → joint1 → ... → shaftEnd
   - **Important**: Skip final segment if last joint is branch type

3. **Project Onto Segments**
   ```typescript
   for (const segment of segments) {
     const projected = projectPointOntoSegment(desiredPosition, segment.start, segment.end);
     const distance = desiredPosition.distanceTo(projected);
     
     if (distance < minDistance) {
       minDistance = distance;
       closestPoint = projected;
     }
   }
   ```

4. **Return Constrained Position**
   - Returns closest point on shaft polyline
   - Clamped to segment endpoints (no extrapolation)

## Recursive Tree Updates

### Problem Statement

When a parent support's joints move:
- Parent shaft geometry changes
- Child branch joints must stay connected to new shaft position
- Grandchild branches must update based on child's new position
- Updates must propagate through entire tree

### Solution: Recursive Update System

**File:** `src/supports/BranchSupports/constraints/updateBranchJoints.ts`

```typescript
export function updateBranchJointsForParent(
  parentSupportId: string,
  allSupports: SupportInstance[],
  visited: Set<string> = new Set()
): Array<{ supportId: string; jointId: string; newPosition: Vec3 }>
```

**Algorithm:**

1. **Find Parent Support**
   ```typescript
   const parentSupport = allSupports.find(s => s.id === parentSupportId);
   ```

2. **Find All Direct Children**
   ```typescript
   const branchSupports = allSupports.filter(s => s.parentBaseId === parentSupportId);
   ```

3. **Update Each Child's Branch Joint**
   ```typescript
   for (const branchSupport of branchSupports) {
     const branchJoint = branchSupport.joints?.find(
       j => j.type === 'branch' && j.lockedToSupportId === parentSupportId
     );
     
     if (branchJoint) {
       // Re-project onto parent's new shaft geometry
       const newPosition = constrainBranchJointToShaft(
         branchJoint.position,
         branchJoint,
         parentSupport
       );
       
       updates.push({ supportId: branchSupport.id, jointId: branchJoint.id, newPosition });
       
       // RECURSIVE: Update this branch's children
       const childUpdates = updateBranchJointsForParent(branchSupport.id, allSupports, visited);
       updates.push(...childUpdates);
     }
   }
   ```

4. **Return All Updates**
   - Flat array of all updates in tree
   - Applied in order by caller

**Infinite Loop Prevention:**
- `visited` set tracks processed supports
- Prevents circular references (shouldn't happen in tree structure)

### Update Timing

**Live Updates (During Drag):**
```typescript
onMove={(delta) => {
  // Update parent joint
  updateJointPositionLive(parentSupportId, jointId, newPosition);
  
  // Update all descendants
  const branchUpdates = updateBranchJointsForParent(parentSupportId, supports);
  for (const update of branchUpdates) {
    updateJointPositionLive(update.supportId, update.jointId, update.newPosition);
  }
}}
```

**History Commit (On Drag End):**
```typescript
onMoveEnd={() => {

  updateJointPosition(parentSupportId, jointId, finalPosition);
  
  // Commit all descendants

    updateJointPosition(update.supportId, update.jointId, update.newPosition);
  }

```

## Branch-to-Branch Support

### Snapping to Branches

The `snapToTrunk()` function handles both trunk and branch parents:

```typescript
for (const s of supports) {
  if (!isTrunk(s) && !isBranch(s)) continue;  // Check both types
  
  // Calculate shaft endpoints
  let shaftStart: Vec3;
  let shaftEnd: Vec3;
  
  if (isBranch(s)) {
    // Find tip joint and branch joint
    const tipJoint = joints.find(j => j.type !== 'branch');
    const branchJoint = joints.find(j => j.type === 'branch');
    
    if (tipJoint && branchJoint) {
      shaftStart = tipJoint.position;
      shaftEnd = branchJoint.position;  // Limit to actual shaft
    }
  } else {
    // Trunk logic...
  }
  
  // Project and snap...
}
```

**Key Points:**
- Branches can be parents too
- Shaft endpoints: tip joint → branch joint (not beyond)
- No phantom projections past branch joint

### Multi-Level Trees

Example structure:
```
Trunk (s1)
  ├─ Branch (s2) → parentBaseId: s1
  │   ├─ Branch (s3) → parentBaseId: s2
  │   └─ Branch (s4) → parentBaseId: s2
  └─ Branch (s5) → parentBaseId: s1
      └─ Branch (s6) → parentBaseId: s5
```

When trunk (s1) moves:
1. Update s2 and s5 (direct children)
2. Update s3 and s4 (children of s2)
3. Update s6 (child of s5)

All updates happen in single frame with live feedback.

## Joint Preview System

### Problem: Phantom Projections

When hovering over a branch in joint creation mode, the preview was projecting beyond the branch joint onto a phantom line.

### Solution: Correct Shaft End Calculation

**File:** `src/components/scene/SceneCanvas.tsx`

```typescript
// Calculate shaft endpoints based on support type
const isBranch = !!support.parentBaseId;

let shaftEnd: { x: number; y: number; z: number };
if (isBranch) {
  // For branches: shaft ends at branch joint
  const branchJoint = support.joints?.find(j => j.type === 'branch');
  if (branchJoint) {
    shaftEnd = branchJoint.position;
  } else {
    shaftEnd = support.base;  // Fallback
  }
} else {
  // For trunks: shaft ends at base top
  shaftEnd = {
    x: support.base.x,
    y: support.base.y,
    z: support.base.z + baseHeight,
  };
}

// Build path: tipEnd -> joints -> shaftEnd
const pathPoints: THREE.Vector3[] = [
  new THREE.Vector3(tipEnd.x, tipEnd.y, tipEnd.z),
  ...joints.map(j => new THREE.Vector3(j.position.x, j.position.y, j.position.z)),
  new THREE.Vector3(shaftEnd.x, shaftEnd.y, shaftEnd.z),
];
```

**Result:**
- Preview sphere only appears along actual shaft
- No floating preview beyond branch joint
- Consistent with actual shaft geometry

## Performance Considerations

### Live Updates vs History

**During Drag:**
- Use `updateJointPositionLive()` - no undo history
- Update every frame for smooth feedback
- Minimal state mutations

**On Drag End:**
- Use `updateJointPosition()` - creates undo entry
- Single commit for entire operation
- Includes all recursive updates

### Optimization: Avoid Console Spam

**Problem:** Debug logging in SupportRenderer caused severe lag
```typescript
// BAD - logs on every render for every support
console.log(`[SupportRenderer] Support ${support.id} has ${joints.length} joints:`, joints);
```

**Solution:** Remove debug logs from render path
- Console.log is expensive (object serialization)
- Hundreds of logs per second during drag
- Causes frame drops and lag

## Edge Cases and Validation

### Missing Joints

If branch support is missing expected joints:
- `snapToTrunk()` skips support (continues to next)
- `constrainBranchJointToShaft()` uses fallback positions
- Prevents crashes, degrades gracefully

### Circular References

Prevented by `visited` set in recursive updates:
```typescript
if (visited.has(parentSupportId)) {
  return updates;  // Stop recursion
}
visited.add(parentSupportId);
```

### Zero-Length Segments

Projection handles degenerate segments:
```typescript
const abLengthSq = ab.lengthSq();
if (abLengthSq === 0) {
  return { x: a.x, y: a.y, z: a.z };  // Return segment start
}
```

## Troubleshooting

### Branch Won't Snap
- Check snap distance (default 5mm)
- Verify parent support has valid shaft
- Ensure mouse within snap threshold
- Check console for errors

### Branch Joint Moves Off Shaft
- Verify `type: 'branch'` is set
- Check `lockedToSupportId` references valid parent
- Ensure constraint function is called in gizmo handler
- Verify parent support exists in supports array

### Recursive Updates Not Working
- Check `parentBaseId` is set correctly
- Verify `updateBranchJointsForParent()` is called
- Ensure function is called for both live and history updates
- Check for circular references (visited set)

### Performance Issues
- Remove debug console.log statements
- Check for unnecessary re-renders
- Verify live updates use `updateJointPositionLive()`
- Profile with React DevTools

### Phantom Projections
- Verify shaft end calculation for branches
- Check joint preview uses branch joint position
- Ensure no extrapolation beyond segment endpoints

## Summary

Branch supports enable efficient, tree-like support structures by:
- Attaching to existing supports instead of build plate
- Using special branch joints that lock to parent shafts
- Updating recursively when parents move
- Supporting multi-level trees (branch-to-branch)
- Providing smooth, constrained movement with live feedback

The system is built on:
- State machine for placement workflow
- Projection algorithms for snapping and constraints
- Recursive updates for tree propagation
- Careful rendering to avoid phantom geometry
- Performance optimization for smooth interaction

This creates a powerful, user-friendly system for creating complex support networks with minimal material usage.
