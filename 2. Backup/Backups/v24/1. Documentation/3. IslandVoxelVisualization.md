# Island Voxel Visualization

## Overview
The Island Voxel Visualization system renders each island as a collection of colored 3D voxels (pixels) based on the pixel-based island detection data. This provides an accurate, performant way to visualize the complete 3D volume of each detected island.

---

## Core Concept

### How It Works

**Island Detection produces pixel grids:**
- Each layer has a grid: `islandLabelsPerLayer[layer]` (Int32Array)
- Each pixel in the grid stores an **Island ID** (0 = no island, 1+ = island ID)
- Grid dimensions: `width × height` pixels at resolution `px_mm`

**Voxel Visualization:**
1. Iterate through all layers
2. For each pixel in each layer's grid
3. If pixel has an island ID > 0:
   - Calculate world position (X, Y, Z) from grid coordinates
   - Create a small cube (voxel) at that position
   - Color the voxel based on its island ID

**Performance:**
- Uses `InstancedMesh` for efficient rendering
- One InstancedMesh per island (allows easy selection/highlighting)
- Thousands of voxels rendered in a single draw call per island

## Color Schemes

### 1. Unique Colors (Default)
- Each island gets a visually distinct color
- Uses golden ratio hue distribution: `hue = (islandId * 0.618033988749895) % 1.0`
- Ensures maximum color separation between adjacent IDs
- Best for distinguishing individual islands

### 2. Lifecycle
- **Green**: Active islands (never merged)
- **Orange**: Merged islands (have a parent)
- Best for understanding merge relationships

### 3. Height Gradient
- **Blue**: Islands at top of model (high Z)
- **Red**: Islands at bottom of model (low Z)
- Gradient based on island's first layer
- Best for understanding vertical distribution

## Features

### Island Selection
- Selected island highlighted in **yellow**
- Selected island opacity increased to 90%
- Integrates with `IslandListCard` selection

### Show/Hide Merged Islands
- Toggle to filter out islands with `parentId !== undefined`
- Helps focus on primary islands only
- Respects parent-child hierarchy from island tracking

### Voxel Sizing
- Voxel size: `grid.px_mm * 0.95` (slightly smaller than pixel size)
- Creates small gaps between voxels for clarity
- Height: `layerHeightMm * 0.95`

## Performance Characteristics

### Memory Usage
- One InstancedMesh per visible island
- Each voxel: 1 matrix (16 floats) + 1 color (3 floats) = 76 bytes
- Example: 10 islands × 10,000 voxels = ~7.6 MB

### Rendering Performance
- Uses GPU instancing (very fast)
- One draw call per island
- Typical: 10-50 draw calls for complex models
- 60 FPS easily achievable with thousands of voxels

### Optimization Opportunities
- Could combine all islands into single InstancedMesh (fewer draw calls)
- Could use LOD (Level of Detail) for distant voxels
- Could cull voxels outside view frustum

## Technical Details

### Coordinate Conversion

**Grid Index to World Position:**
```typescript
const row = Math.floor(pixelIndex / grid.width);
const col = pixelIndex % grid.width;

const worldX = grid.originX + col * grid.px_mm;
const worldY = grid.originZ + row * grid.px_mm;  // Note: Y maps to Z in grid
const worldZ = layer * layerHeightMm;
```

```typescript
const matrix = new THREE.Matrix4();
for (let i = 0; i < positions.length; i++) {
  matrix.setPosition(positions[i]);
```

## Advantages Over Mesh Painting

### Why Voxels Instead of Vertex Colors?

**Voxel Approach (Current):**
- ✅ 100% accurate to pixel detection data
- ✅ No ambiguity at triangle boundaries
- ✅ Easy to implement and debug
- ✅ Clear visual separation between islands
- ✅ Works perfectly with pixel-based detection
- ❌ Blocky appearance (not smooth)

**Vertex Painting Approach (Previous):**
- ✅ Smooth mesh surface
- ❌ Complex sampling strategy needed
- ❌ Triangles spanning multiple islands cause issues
- ❌ Color bleeding at boundaries
- ❌ Doesn't match pixel data exactly

**Conclusion:** Voxels are the natural visualization for pixel-based island detection. They directly represent the underlying data structure.

## Summary

The Island Voxel Visualization system provides:
- **Accurate** representation of pixel-based island detection
- **Performant** rendering using GPU instancing
- **Flexible** color schemes for different analysis needs
- **Interactive** selection and filtering
- **Clear** visual separation between islands

This approach leverages the existing `islandLabelsPerLayer` data structure to create a direct, unambiguous visualization of detected islands.
