# Island Detection Method

## Overview

This document explains how the STL slicer detects and tracks unsupported regions (islands) in 3D models. The system identifies areas that would fail to print properly without support structures by analyzing the model layer-by-layer.

---

## What is an Island?

**An island is the entire 3D volume of model material that sits on unsupported regions.**

Think of it like a tree growing from a tiny seed:
- The seed (unsupported base) triggers the island detection
- The entire tree above it (all connected geometry) is part of that island
- The island continues upward until it merges with another island or reaches the top of the model

### Key Concept

Islands are **NOT** just the unsupported pixels at a single layer. They are **3D volumes** that:
1. Start when unsupported geometry is detected
2. Include all connected model material vertically above the unsupported base
3. Continue through multiple layers as the geometry grows, shrinks, or changes shape
4. End when they merge with another island or the scan completes

### Example: Cone Tip

A cone starting from a mathematical point:
- **Layer 0**: Single pixel (unsupported) → Island detected
- **Layer 1-50**: Cone grows larger → Same island continues
- **Entire cone** = One island (not 50 separate islands)

## How It Works (Simple Explanation)

### Step 1: Slice the Model

The model is divided into horizontal layers (like slicing bread). Each layer represents a thin cross-section of the model at a specific height.

**Default layer height**: 50 microns (0.05mm)

### Step 2: Convert to Pixels

Each layer's cross-section is converted into a grid of pixels (like a black-and-white image):
- **Black pixels** = Empty space
- **White pixels** = Solid model material

**Default pixel size**: 0.10mm (100 microns)

This creates a 2D bitmap representation of each layer.

### Step 3: Find Unsupported Areas

For each layer, the system compares it to the layer below:

1. **Expand the previous layer** by a "support buffer" distance (default: 0.6mm)
   - This simulates how far material can bridge without support
   
2. **Compare current layer to expanded previous layer**
   - Pixels in the current layer that DON'T overlap with the expanded previous layer are **unsupported**
   
3. **Group connected unsupported pixels**
   - Adjacent unsupported pixels are grouped into components (potential islands)

### Step 4: Track Islands Across Layers

The system follows islands as they grow upward through the model:

- **New island**: Unsupported pixels with no connection to previous layer
- **Continuation**: Pixels that overlap with an existing island from below
- **Merge**: When multiple islands join together into one

Each island gets a unique ID that stays with it throughout its lifetime.

### Step 5: Handle Merges Intelligently

When islands merge, the system needs to determine which one is the "parent":

1. **Wait 30 layers** after the merge
2. **Track which original island** the merged geometry overlaps with most
3. **Assign parent** based on maximum overlap
4. Mark other islands as "children" of the parent

This ensures accurate parent-child relationships even for complex geometries.

### Step 6: Calculate Volumes

For each island, the system calculates:
- **Volume**: Sum of (area × layer height) for all layers
- **Maximum cross-sectional area**: Largest area across all layers
- **Layer span**: First and last layer where the island appears

### Step 7: Filter Small Islands

Islands smaller than a minimum threshold are removed:
- **Default threshold**: 0.01 mm² maximum cross-sectional area
- This filters out noise and tiny features that don't need support

## Technical Details

### Coordinate System

- **Z-axis**: Vertical (height) - points upward
- **Y-axis**: Forward/backward (depth)
- **X-axis**: Side-to-side (width)

Layers are sliced at Z heights, creating horizontal XY plane cross-sections.

### Rasterization Process

**Input**: Closed polygon loops from mesh cross-section
**Output**: Binary pixel grid (Mask)

The rasterization algorithm:
1. Computes bounding box of all loops
2. Creates grid with specified pixel size (px_mm)
3. For each pixel center, tests if point is inside any polygon using ray-casting
4. Stores result as 1 (solid) or 0 (empty)

**Point-in-polygon test**: Uses ray-casting algorithm with horizontal ray, counts intersections with polygon edges.

### Morphological Dilation

To find supported regions, the previous layer is expanded (dilated) by the support buffer distance.

**Algorithm**: Circular dilation
- For each pixel in output grid
- Check all pixels within radius (support_buffer_mm / px_mm)
- If any source pixel is solid, output pixel is solid
- Uses distance check: `dx² + dy² ≤ radius²`

**Purpose**: Simulates material bridging capability - current layer is "supported" if it's within the buffer distance of previous layer material.

### Component Labeling

Identifies connected regions of unsupported pixels using flood-fill.

**Algorithm**: Breadth-first search (BFS)
- Scan grid for unlabeled solid pixels
- Start new component with unique ID
- Flood-fill to all connected neighbors (4-connectivity or 8-connectivity)
- Track component metadata (ID, pixel count, area)

**Connectivity options**:
- **4-connect**: Only horizontal/vertical neighbors (default)
- **8-connect**: Includes diagonal neighbors

### Island Tracking (IslandTracker)

Manages stable island IDs across layers and parent-child relationships.

**Core data structure**:
```typescript
interface Island {
  id: number;                          // Unique stable ID
  firstLayer: number;                  // Layer where island first appeared
  lastLayer: number;                   // Last layer (updated as island grows)
  status: 'active' | 'complete';       // Active = still growing, Complete = ended
  perLayerAreaMm2: Map<number, number>; // Area at each layer
  totalAreaMm2: number;                // Sum of all layer areas
  volumeMm3: number;                   // Calculated volume
  maxAreaMm2: number;                  // Largest cross-section
  parentId?: number;                   // Parent island (if merged)
  childIds: number[];                  // Child islands (that merged into this)
}
```

**Processing logic per layer**:

1. **Find solid components** in current layer (all connected solid pixels, not just unsupported)

2. **For each component, check overlap with previous layer**:
   - No overlap → New island (assign new ID)
   - Overlaps one island → Continuation (inherit that ID)
   - Overlaps multiple islands → Merge (create pending merge)

3. **Flood-fill from unsupported seeds** to label ALL connected solid pixels with island ID
   - This captures the full 3D volume, not just the unsupported base

4. **Track pending merges**:
   - Mark merging islands as 'complete' at layer before merge
   - Create temporary merged island placeholder
   - Track overlap counts for next 30 layers
   - Determine true parent based on maximum overlap
   - Transfer evaluation window layers to parent island

### Cross-Section Caching

To optimize performance, cross-sections are cached:
- Each layer's cross-section is computed once
- Reused as "previous layer" for next layer
- Cache keeps only last 3 layers to avoid memory bloat
- Reduces `computeLoopsAtZ` calls by ~50%

### Parallel Processing

Workers scan layers in parallel:
- Uses Web Workers (one per CPU core)
- Each worker processes layers independently
- Results collected and processed sequentially by IslandTracker
- Typical speedup: 2-4x depending on CPU cores

**Worker responsibilities**:
- Rasterize current and previous layer loops
- Compute supported regions (dilation + comparison)
- Label unsupported components
- Return masks, labels, and component metadata

**Main thread responsibilities**:
- Coordinate workers
- Sequential island tracking (must be sequential for stable IDs)
- Volume calculation
- Filtering and cleanup

### Volume Calculation

Volume is calculated from per-layer areas:

```
Volume = Σ (area_at_layer × layer_height)
```

**Why use perLayerAreaMm2**:
- Recorded at time of layer processing (before any relabeling)
- Accurate even after merges and parent reassignment
- Each island's volume reflects its actual geometry

**Alternative approaches NOT used**:
- Counting pixels in final labels (inaccurate after merges)
- Voxel counting (memory intensive)

### Filtering Strategy

**Post-scan volume-based filtering**:
- ALL islands tracked during scan (no size checks during detection)
- After scan completes, filter by maximum cross-sectional area
- Remove filtered island IDs from pixel labels

**Why filter by area instead of volume**:
- Area is more relevant for 3D printing
- Small base area = won't print reliably, regardless of volume
- Handles growing features (cone tips) correctly

**Default threshold**: 0.01 mm² (0.1mm × 0.1mm square)

### Placeholder Resolution

Merged islands create temporary placeholders during evaluation:
- Placeholders marked with `isMergedPlaceholder: true`
- After evaluation, placeholders assigned to true parent
- Pixel labels updated to resolve placeholder chains
- Example chain: `#25 → #23 → #19 → #16 → #1` (true parent)

**Resolution algorithm**:
- Build map of placeholder → parent
- For each pixel, follow chain to find non-placeholder parent
- Update pixel label to true parent ID
- Remove placeholder islands from final results

## Configuration Parameters

### Layer Height
- **Default**: 50 microns (0.05mm)
- **Range**: 10-200 microns typical
- **Effect**: Smaller = more accurate, slower scan

### Pixel Size (px_mm)
- **Default**: 0.10mm (100 microns)
- **Range**: 0.05-0.50mm typical
- **Effect**: Smaller = more accurate, more memory, slower

### Support Buffer
- **Default**: 0.6mm
- **Range**: 0.1-2.0mm typical
- **Effect**: Larger = fewer islands detected (more bridging allowed)

### Connectivity
- **Default**: 4-connect
- **Options**: 4-connect or 8-connect
- **Effect**: 8-connect finds fewer, larger components

### Minimum Island Area
- **Default**: 0.01 mm²
- **Range**: 0.0001-10 mm²
- **Effect**: Larger = fewer small islands shown

## Performance Characteristics

### Typical Scan Times
- **Small model** (20mm cube): 1-2 seconds
- **Medium model** (100mm tall): 5-10 seconds
- **Large model** (200mm tall): 15-30 seconds

### Memory Usage
- **Grid size**: width × height × num_layers × 4 bytes
- **Example**: 1000×1000 grid, 2000 layers = 8 GB
- **Optimization**: Cross-section caching keeps only 3 layers

### Bottlenecks
1. **Cross-section computation**: Slicing mesh at each Z height
2. **Rasterization**: Point-in-polygon tests for each pixel
3. **Component labeling**: Flood-fill operations

### Optimization Strategies
- Parallel worker processing (CPU cores)
- Cross-section caching (50% reduction)
- Efficient flood-fill with pre-allocated queues
- Typed arrays (Int32Array, Uint8Array) for performance

---

## Algorithm Complexity

- **Rasterization**: O(W × H × L) where W=width, H=height, L=loops
- **Dilation**: O(W × H × R²) where R=radius in pixels
- **Component labeling**: O(W × H) per layer
- **Island tracking**: O(N × M) where N=components, M=previous islands
- **Overall**: O(layers × width × height) - linear in grid size

## Edge Cases Handled

### Cone Tips (Growing from Point)
- Start with 0-1 pixels (mathematical point)
- Grow larger in subsequent layers
- **Solution**: No per-layer filtering, only post-scan filtering by total volume

### Merging Islands
- Multiple islands join into one
- **Solution**: Pending merge evaluation over 30 layers to determine true parent

### Splitting Islands
- One island splits into multiple
- **Solution**: Treated as parent completing, new children starting

### Floating Islands
- Islands with no connection to previous layer after layer 0
- **Solution**: Detected as new islands, tracked independently

### Thin Bridges
- Material that barely connects to layer below
- **Solution**: Support buffer determines if connection is sufficient

## Output Data

### ScanResults Object
```typescript
{
  grid: GridRef;                    // Grid dimensions and origin
  layers: ScanLayerResult[];        // Per-layer masks and counts
  islands: Island[];                // All tracked islands
  islandLabelsPerLayer: Int32Array[]; // Per-layer pixel labels (island IDs)
  firstHit: Int16Array;             // First layer each pixel appears
  lastHit: Int16Array;              // Last layer each pixel appears
  baseFootprint: Uint8Array;        // Binary mask of island bases
  baseLabels: Int32Array;           // Component labels for bases
  compBase: Int16Array;             // First layer per base component
  compTop: Int16Array;              // Last layer per base component
}
```

### Island Data
Each island includes:
- Unique stable ID
- First and last layer
- Status (active/complete)
- Per-layer areas (Map)
- Total area and volume
- Maximum cross-sectional area
- Parent and child relationships

### Pixel Labels
- **islandLabelsPerLayer[layer][pixel]** = Island ID (0 = not island)
- Used for 3D visualization (voxels, vertex painting)
- Used for volume queries and analysis
